# Phosphene 1.0 产品规格

> 本文是 Phosphene 1.0 的冻结规格。第一版按正式产品标准建设，不做临时雏形；开发阶段只是施工顺序，不代表残缺版本可以发布。

## 产品定位

Phosphene 是一套“一位用户 + 一位 AI”的私人任务、积分、奖励与陪伴系统。每位使用者独立部署一套实例：

- 不做多租户、公开注册或组织体系。
- 代码内部只使用 `AI`、`user`、`system`。
- 显示称呼可由用户自定义。
- AI 通过 MCP 连接，用户通过网站操作。
- 数据、图片与服务均归部署者自己所有。

## 生产架构

- Node.js LTS + TypeScript
- Express + React + Vite，单域名提供网站、REST 和 MCP
- 官方 MCP TypeScript SDK，Streamable HTTP `/mcp`
- Drizzle ORM + 嵌入式 SQLite，数据位于持久化卷 `/data`
- 默认使用 `/data` 内的私有图片存储
- Sharp 对上传图片重新解码、清除 EXIF 并生成审核预览
- Docker；Zeabur 默认只部署一个 Git 服务并挂载一个 `/data` 卷

## 角色与权限

### user

- 登录网站、查看任务、提交文字与图片证据
- 自我确认允许自证的任务
- 查看积分、统计、历史和成就
- 兑换奖励
- 设置边界、称呼、时区、惩罚开关和每日扣分上限
- 生成、轮换和撤销 AI MCP Token
- 导出与恢复数据

### AI

- 创建、编辑、取消和判定任务失败
- 审核需要 AI 确认的提交
- 创建或管理奖励、履行兑换
- 查询任务、历史与概览
- 在用户边界和每日限额内奖励或扣除积分
- 无权修改用户密码、边界或主动兑换

### system

- 生成每日重复任务实例
- 处理逾期与扣分
- 结算任务积分、连击、统计和成就
- 维护不可重复的幂等事件与审计日志

## 任务

任务类型：

- `daily`
- `challenge`
- `surprise`

仅 `daily` 可设置 `recurrence: once | daily`。重复 daily 有开始日期、可选结束日期、每日截止时间、暂停与恢复能力；数据库以 `task_series` 保存规则，以 `tasks` 保存每日独立实例。Challenge 与 surprise 固定为一次性。

Surprise 默认在用户下次访问网站时揭晓，也可以指定展示时间。

任务状态：

- `pending`
- `submitted`
- `completed`
- `failed`
- `expired`
- `cancelled`

确认模式：

- `self`
- `ai_review`

证据要求：

- `none`
- `text`
- `image`
- `text_or_image`
- `text_and_image`

用户在截止时间前提交但 AI 延迟审核时，完成日期仍归属于用户提交的当地自然日。

## 图片证据

- 每次提交最多 4 张
- 单张最多 10 MB
- JPEG、PNG、WebP
- 校验真实文件格式与最大像素
- 重新解码并移除 EXIF/GPS
- 生成 AI 审核预览
- 默认写入 `/data/uploads` 私有目录，不公开原始 URL
- 数据库只保存对象 Key、MIME、大小、SHA-256、宽高和归属

## 积分

- easy ×1、medium ×2、hard ×3
- 任务奖励 = 基础积分 × 难度倍率
- 失败或逾期扣最终积分的 50%
- 余额默认不低于 0
- 账本事件不可修改，只能通过 correction 调整
- 结算、扣分和兑换必须原子化且具备幂等键

流水类型：

- `task_reward`
- `streak_bonus`
- `task_penalty`
- `redemption`
- `manual_bonus`
- `manual_penalty`
- `correction`

## 连击与统计

按用户时区计算自然日。网站提供九个常见地区选项，不要求用户手写 IANA 标识。当天至少完成一个任意类型任务即为有效完成日；一天没有完成任何任务，跨过午夜后当前连击归零，即使当天没有被分配任务。

每天只发一次连击奖励：

- 第 1 天：0
- 第 2～5 天：+1
- 第 6～7 天：+2
- 第 8 天起：+3

长期统计：

- 当前余额、累计获得、累计消费、累计扣除
- 当前连击、最长连击
- 总坚持天数（有完成记录的不同自然日）
- 总完成任务数
- 各任务类型和难度完成数
- 今日完成状态、待完成、待审核、待履行数量

## 奖励

用户从网站兑换，AI 不能替用户消费。兑换读取服务端奖励价格并在同一事务内检查余额、创建待履行记录和立即扣除积分；这不等于奖励已经执行。AI 可以通过 `manage_rewards` 创建、修改、归档自定义奖励，并在实际兑现后标记履行。

预设奖励：

- 指定 AI 写东西：15 积分
- “AI 听你的”券：20 积分

上述预设中的 `AI` 在网站和 MCP 返回值中按用户设置的显示称呼替换。被用户或 AI 自定义过名称的项目保持原文，不做字符串改写。

## MCP 工具（固定 7 个）

1. `create_task`
2. `query_tasks`
3. `manage_task`
4. `get_overview`
5. `query_history`
6. `manage_rewards`
7. `adjust_points`

用户提交、上传、兑换、设置等行为通过网站 REST API 完成，不增加 MCP 工具。

## 网站

正式 1.0 包含：

- 首次设置、登录
- 仪表盘、今日任务、全部任务、任务详情
- 文字和图片提交
- 积分、连击与完整统计
- 奖励商城与兑换历史
- 成就、活动历史
- 设置、边界、称呼、时区
- AI MCP 连接与 Token 轮换
- 数据导出与恢复
- 可安装 PWA、独立窗口、移动端主屏幕图标与安全区适配

## 安全

- 默认由首位成功提交设置的人认领实例；初始化完成后入口永久关闭
- 可选 Setup Token 保护，适合已经公开或无法立即认领的部署
- Argon2id 密码哈希
- Secure / HttpOnly / SameSite 会话 Cookie
- CSRF 防护、速率限制、严格参数校验
- AI Token 仅显示一次，数据库只存哈希，可轮换和撤销
- 用户边界只能由 user 修改，AI 只读
- 上传图片重新编码，防止伪造文件和元数据泄漏
- 所有写操作记录审计日志

## 内置成就

- 第一次完成任务
- 连击 3 / 7 / 14 / 30 / 100 天
- 总坚持 7 / 30 / 100 / 365 天
- 完成 10 / 50 / 100 / 500 个任务
- 第一次 hard；完成 10 / 50 个 hard
- 第一次 challenge、第一次 surprise
- 累计获得 100 / 500 / 1000 / 5000 积分
- 第一次兑换、完成 10 次兑换

## 发布门槛

- 一个 Git 服务与一个 `/data` 持久化卷即可获得完整网站和 MCP 域名
- 重启不丢数据、不重复生成 daily
- 图片私有并清除定位元数据，AI 可审核预览
- 网络重试不重复发分、扣分或兑换
- 延迟审核不错误中断连击
- 时区和午夜边界有自动测试
- 数据与图片支持导出、恢复和备份
- MCP 工具恰好 7 个
- 核心领域逻辑有单元与集成测试
- 网站手机与桌面均可用
- PWA manifest、图标和离线应用壳层可用；私有 API 与图片不进入离线缓存
- README 可让陌生用户独立完成部署与连接
